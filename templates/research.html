Research & Methodology

This page summarizes the academic and technical thinking behind the system, from problem definition to trained model deployment.

Problem Statement

Cyber threats continue to rise while many users and organizations still lack a quick, structured way to estimate their current exposure. This project addresses that gap using data-driven risk classification.

Objectives

  • Predict risk level using security and awareness indicators.
  • Provide actionable recommendations after prediction.
  • Promote cybersecurity awareness through supporting content.

Why this branch was selected

  • The output of this project is categorical, so classification is more suitable than regression.
  • Random Forest handles tabular security data well and is robust against overfitting.
  • The selected path supports explainable decision-making for project demonstration.

Final workflow used in this project

  • Collect cybersecurity dataset features from structured records.
  • Choose the core model inputs used by the predictor.
  • Train the Random Forest classifier on labeled cybersecurity risk data.
  • Predict low, medium, or high risk levels inside the Flask application.

Why Random Forest?

Random Forest is robust on mixed numerical features, reduces overfitting through ensemble learning, and remains easier to explain than many black-box alternatives.

Evaluation Snapshot

The current rebuilt model was trained on 5,000 records and achieved approximately 89.1% accuracy on the holdout split.
