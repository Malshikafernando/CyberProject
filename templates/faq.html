Frequently Asked Questions

These answers help explain the project clearly during demonstrations, reports, and viva discussions.

What is cybersecurity risk?

Cybersecurity risk is the possibility that a threat exploits a weakness and causes loss, disruption, or unauthorized access.

How does this system make predictions?

The trained model evaluates 10 input features related to security controls, awareness, vulnerabilities, and monitoring, then predicts a low, medium, or high risk class.

Why combine awareness with technical controls?

Real-world incidents often happen because technical defenses and user behavior interact. This system models both dimensions instead of treating them separately.

Does the application permanently store user data?

No. Input values are processed for prediction and the current result is kept only in session memory for the active user flow.

Can the model be retrained?

Yes. The project includes a training script that rebuilds the model from the Excel dataset and saves a fresh `cyber_model.pkl` file.
